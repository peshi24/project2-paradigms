# project2-paradigms
create a prolog backend that will allow a user to prepare a work schedule based on the facts given

# description

This project implements a workstation scheduling system in Prolog that generates valid employee schedules for morning, evening, and night shifts while satisfying a variety of staffing and employee constraints. The program ensures that every employee is assigned to exactly one shift, checks for workstation staffing limits, checks for restricted workstations and shifts, and checks for idle workstations. The scheduling system uses recursive predicates, subset generation, and constraint validation to construct valid schedules through Prolog backtracking.

The project contains three main files. The backend.pl file contains the primary scheduling logic, including predicates for shift assignment, workstation assignment, employee validation, subset generation, and testing predicates used to test and debug the code. The testing.pl file contains all employee facts, workstation definitions, workstation constraints, and shift restrictions used by the scheduler, in this case same as the example_input_1 given. 

To run the program, first open a terminal and navigate to the project directory. Start SWI-Prolog by running swipl in the terminal. Once inside SWI-Prolog, consult the backend file using ?- [backend, testing]. This will load the backend.pl and testing.pl file manually. MAKE SURE that the files are in the same directory. To generate a valid schedule, run the query ?- plan(Plan). Additional valid schedules can be generated by entering ; after a solution appears. Several testing predicates are also included to verify schedule correctness, such as checking for unassigned employees, duplicate shift assignments, workstation restrictions, and shift restrictions.

Before grading, please note that the scheduler may generate multiple valid schedules because the program relies on Prolog backtracking. The project was developed and tested using SWI-Prolog 9.x and requires the library(lists) module for predicates such as member/2 and subtract/3.
